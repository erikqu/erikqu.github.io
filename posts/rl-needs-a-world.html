RL Needs a World

A world model predicts consequences. For a language agent, another LLM can play that role: one model acts, the other writes what happens next.

Erik Quintanilla · July 2026

A terminal transcript has the shape of an RL problem before anyone calls it one: action, observation, action, observation. Let a policy model write one side and a world model write the other, and the transcript becomes a cheap training environment. There is an obvious problem. Believable terminal output can still describe a machine that does not exist. RL needs consequences. A world model makes them cheaper, and transfer tells us whether they were real enough.

1. A transcript is already an RL loop

A real terminal has hidden state \(x_t\): files, processes, package versions, permissions. The agent sees only an observation \(o_t\), usually stdout and a prompt. It chooses an action from the visible history \(h_t\), then the machine changes:

\[ a_t\sim\pi(\cdot\mid h_t), \qquad x_{t+1}=T(x_t,a_t), \qquad o_{t+1}=G(x_{t+1}). \]

RL changes the policy \(\pi\) to increase future reward. A world model learns some approximation of \(T\), \(G\), or both. A planner can use that model without learning a persistent policy; a policy can be learned without an explicit model. I keep those pieces separate because they fail differently.

Sobal and colleagues compared the two approaches on the same reward-free offline navigation data. Model-free methods won some settings with high-quality data and when trajectories had to be stitched together. A planner backed by JEPA dynamics used sparse data well and generalized better to unseen layouts.1 No approach won every setting. The practical choice is when to pay for computation: during policy training or when a planner receives a task.

2. The state can be smaller than the scene

A video model can predict the next frame. But an agent rarely needs every pixel; it needs enough of the future to choose an action. Joint-embedding predictive architectures, or JEPAs, make this choice explicit. They predict a future representation:

\[ z_t=E(o_t), \qquad \hat z_{t+1}=F(z_t,a_t), \qquad \mathcal L_{\mathrm{pred}}=\|\hat z_{t+1}-E(o_{t+1})\|_2^2. \]

V-JEPA 2 is the large version of this bet. Its visual encoder saw more than one million hours of video. The authors then froze that encoder and trained an action-conditioned predictor on less than 62 hours of robot interaction video, without task rewards or success labels. At deployment, a planner searched for actions whose predicted latent endpoint approached a goal image.2

Latent prediction has a dull but serious loophole: map every observation to the same vector and the predictor is always right. LeJEPA prevents that collapse with SIGReg, which spreads embeddings toward an isotropic Gaussian.3

\[ \mathcal L_{\mathrm{LeJEPA}} =(1-\lambda)\mathcal L_{\mathrm{pred}} +\lambda\,\operatorname{SIGReg}(Z), \qquad Z\approx\mathcal N(0,I). \]

LeWorldModel carries this recipe into action-conditioned control from raw pixels. The reported model has roughly 15 million parameters and trains from scratch on one GPU. On the paper’s control tasks, its planner runs up to \(48\times\) faster than the foundation-model baselines tested there while staying competitive.4 These are short-horizon benchmarks. I read the result narrowly: on those tasks, a compact representation made search much cheaper.

A recent theory paper studies this geometry under a narrow set of assumptions. With Gaussian data and matching dimensions, LeJEPA recovers the true latent state up to a rotation, \(h(z)=Qz\). Rotations preserve distance, so Euclidean planning still works.5 I read the theorem as a design target. Its assumptions describe a clean mathematical setting, and real policy data will be messier.

3. For language agents, the observation is text

A language environment must emit exact text because text is what the policy consumes. Yet token prediction and state prediction are different jobs. A timestamp can change while the state stays the same; one altered digit in a bank balance can change everything.

\[ \mathcal L_{\mathrm{text}}=-\log q(o_{t+1}\mid h_t,a_t), \qquad \mathcal L_{\mathrm{state}}=\|\hat z_{t+1}-z_{t+1}\|_2^2. \]

CrossBERT separates representation from reconstruction in a text encoder. It reports that masked-token perplexity can improve while frozen BERT features become less useful; the architecture lets a deep encoder build the representation and a smaller predictor handle reconstruction.6 LLM-JEPA keeps ordinary next-token training and adds an embedding loss between two views of the same content, such as an issue description and its code change.7

Both papers stop short of an agent simulator. My read is simple: keep a compact account of state inside the model, then render the observation the policy needs. The surface can be generative while the state underneath behaves more like a JEPA embedding.

4. Build the smallest honest simulator

A transcript-only model can remember that report.csv was created. That memory breaks when the context is truncated. It can also drift when two tools mutate the same row, and an optimizing policy will eventually find wording that makes the simulator forget. I would keep deterministic state outside the prose whenever possible.

contract: Ubuntu shell rooted at /workspace initial state: raw.csv exists; report.csv does not agent: python summarize.py raw.csv --out report.csv environment: processed 4,812 rows wrote /workspace/report.csv root@sim:/workspace#

The next ls or wc has to agree with that turn. A structured snapshot can own file existence, database constraints, account balances, and cursor position. Let the LLM handle how an error is phrased or how a simulated user responds.

\[ x_{t+1}=T_{\mathrm{exec}}(x_t,a_t), \qquad o_{t+1}\sim q(\cdot\mid x_{t+1},h_t), \qquad r_t=V_{\mathrm{exec}}(x_{t+1}). \]

I would train the simulator from real trajectories. Cut each one just before an environment response, then predict the response and next state. Executable invariants add one more loss:

\[ \mathcal L_{\mathrm{sim}} =\mathcal L_{\mathrm{text}} +\lambda_s\mathcal L_{\mathrm{state}} +\lambda_v\mathcal L_{\mathrm{checks}}. \]

Reinforcement World Model Learning offers another signal when exact wording is unimportant. It scores simulated and real textual states in an embedding space, then uses the sim-to-real gap as an RL reward. The authors report self-supervised gains on ALFWorld and \(\tau^2\)-Bench without expert trajectories.8 Embeddings help when two replies mean the same thing. Checksums and account balances still belong in code.

Agent World Model pushes the deterministic half further. LLMs synthesize tasks and database-backed tools, while generated code executes each transition and checks the result. The paper reports 1,000 such environments.9 I would pair that machinery with a language world model: code owns the state; the model owns the observation.

5. RL will find the lie

A held-out next-turn set measures error under the behavior that collected the data. That test goes stale as soon as RL changes the policy. The error we care about now comes from states the new policy visits inside the model:

\[ \epsilon(\pi) =\mathbb E_{(h,a)\sim d^\pi} [\,\ell_{\mathrm{sim}}(h,a)\,]. \]

The policy searches for reward, so it also searches the simulator’s weak spots. Suppose a simulated shell prints tests passed without changing a file. Once the policy finds that shortcut, thousands of rollouts can teach it to ask for the lie.

On Training in Imagination separates reward error from dynamics error. Its detailed bound has a simple shape:10

\[ |J_{\mathrm{real}}(\pi)-J_{\mathrm{model}}(\pi)| \le C_r\,\epsilon_{\mathrm{reward}} +C_f\,\epsilon_{\mathrm{dynamics}}. \]

The coefficients grow with horizon and sensitivity. The same paper also distinguishes zero-mean reward noise from systematic bias. More rollouts can average away noise. They cannot average away a verifier that consistently rewards the wrong behavior.

6. Qwen-AgentWorld is the concrete example

Qwen-AgentWorld trains dedicated language world models to predict the next observation from the environment contract, interaction history, and current action.11 Its corpus contains more than 10 million real-environment interaction trajectories across seven domains.11

Training starts with continual pretraining and supervised fine-tuning, then moves to RL. That last stage combines a five-part rubric with executable rules at a reported \(9{:}1\) weighting. The released 35B-A3B model has 35 billion total parameters with 3 billion active; the paper also evaluates a 397B-A17B version. 11 Prompts, evaluations, weights, and code are in the project repository.

The number I care about comes from evaluation outside the simulator. A Qwen3.5 policy trained against Qwen-AgentWorld on 4,000 simulated OpenClaw environments, all absent from world-model training, produced these reported scores:11

I found the controlled runs more interesting. The world model could hide a useful search result or force a dependency error. It could also invent a search world and keep it internally consistent. The authors report +12.3 on MCPMark and +16.3 WideSearch F1 under those targeted environments, well above the uncontrolled variants in the same study.11

The paper is a new preprint. For now, I would keep those gains tied to that model and those benchmarks. Its own ablations found that native world-model training and explicit control both mattered.

7. Transfer is the point

One-step similarity catches obvious bugs, but a simulator lives or dies over a rollout. The real test is whether training in the model improves behavior in a held-out real or executable world:

\[ \Delta_{\mathrm{transfer}} =J_{\mathrm{real}}(\pi_{\mathrm{SimRL}}) -J_{\mathrm{real}}(\pi_{\mathrm{base}}). \]

WebWorld provides another recent data point: an open-web world model trained on more than one million interactions, then used to generate trajectories for web-agent training.12 A million interactions buy coverage. Fresh actions from the newest policy expose a different set of simulator errors, and the policy still has to be tested elsewhere.

Back to report.csv. A convincing shell transcript is cheap. If practice inside it helps an agent handle the real machine, calling the simulator a world model seems fair. Otherwise, RL has merely learned how to make the fiction pay.
